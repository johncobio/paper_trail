writeup — finding M or P given only L

CSE 5381 / A3 / spring 2026

the question

the assignment asks how someone could find M or P given only L (the periodicity). the attacker is assumed to have the stego file P' (the modified carrier), and knows the period L but not the starting skip S, not the mode C, and not the original P or M.

what knowing only L gives you

not a lot on its own. L only tells you the spacing between carrier bits that got overwritten. the attacker still has to guess:

  • S, the starting bit offset (typically 0..a few thousand for common file formats, so probably < 2^16 candidates if you assume a reasonable upper bound)
  • C, the mode (only a handful in this implementation, so basically free)
  • the payload itself, which could be anything

attacks

1. brute force on S and C

since C is small and S is bounded by the carrier size, the attacker can just try each (S, C) pair, read the first 32 bits at the implied positions, and treat them as the message-length header. most guesses produce a "length" that is either larger than the carrier or is clearly nonsense (e.g. length bytes where the high-order byte is non-zero but the carrier is 2MB). only plausible lengths survive, and those are much fewer than the raw S * |C| space.

after filtering, the attacker reads the candidate M and runs sanity checks: does it start with a known magic number (FF D8 for JPEG, %PDF, PK\x03\x04 for zip, UTF-8 text with reasonable entropy, etc.). this is very effective when M has structure.

2. use the carrier's own format as a crib

if P is a known format (PNG, BMP, WAV), the attacker knows which bits inside a byte can vary without breaking the file (the LSBs of image pixel bytes, for instance). looking at P', the locations where LSBs differ from what a natural image would have stand out. with a period L you'd specifically look at every L-th bit and test whether those bits have a different distribution (e.g. closer to 50/50 random) than surrounding bits. chi-square on bit positions modulo L is a one-liner detector.

3. two-file attack (known/chosen carrier)

if the attacker can get both P' and a copy of the original P (e.g. the sender used a stock photo), the bits that differ are the message bits, in order. the attacker doesn't even need L: they XOR P ⊕ P', take every non-zero-bit position, and those bits are M. L only matters if P is unknown.

4. structural holes in the scheme

with C=0 (flat mode), the set of touched bits is an arithmetic progression. this is a strong fingerprint. any periodic anomaly at spacing L screams "stego here". modes 1–3 help a little because they break periodicity, but they do it deterministically — once C is guessed, the positions are fixed. a real defense would need a keyed, data-dependent schedule (e.g. HMAC-drive position PRNG), which this assignment's scheme does not have.

finding M specifically

in short: L alone narrows the attack a lot but doesn't solve it. the attacker still has to search S and C, and recognize M by its format or statistics. if M is encrypted before embedding, finding it even with all of S, L, C is just as hard as breaking the cipher — which is the standard recommendation: always encrypt M before stego.

finding P specifically

P is only recoverable up to the bits that were overwritten. with L, the attacker knows which bits (once they guess S, C) are no longer original — those are lost. if P came from a known source (a public image, a known audio file), the attacker can just look up the original. otherwise, the 1/L fraction of bits that were rewritten are gone.
