paper_trail

about this thing

programming assignment for CSE 5381/4381 (Information Security II, spring 2026). it's a small web app that hides a message inside a carrier file using bit-periodic steganography, and shows the result on a public board.

how the embedding works

given a carrier P, a message M, a skip S, a period L, and a mode C:

  1. prepend a 32-bit big-endian length to M so the extractor knows where to stop. call the whole thing the payload.
  2. walk over P bit by bit starting at bit index S. every step, overwrite P's current bit with the next payload bit and jump ahead by step(i, L, C).
  3. the step function depends on mode:
    • C=0 flat — step is always L
    • C=1 ramp — step cycles through [L, 2L, 4L, 2L]
    • C=2 fib — step = L * fib(i)
    • C=3 prime — step = L * prime(i)
  4. extracting is the exact same walk, but we read the bits at those positions instead of writing them. the 32-bit length prefix tells us when to stop.

capacity

with flat mode the carrier fits about (LenP - S) / (8 * L) bytes. with the other modes the step grows faster so capacity shrinks — fib and prime modes are meant for short messages in a large carrier.

security caveats

this is a class demo. it is not strong steganography:

  • raw LSB-style overwrites on a JPEG/PNG byte stream will corrupt the image structure. for image carriers, pick S that lands after the header and use a format whose byte stream tolerates random flips (raw .bmp or .wav work best). for the board preview, smaller L and an uncompressed format keep the image viewable.
  • statistical tests can detect LSB-style modulation pretty easily. there's no attempt to match carrier statistics.
  • the auth is localStorage-only because GitHub Pages can't run a backend. see the README for how i'd do it differently with a real server.
